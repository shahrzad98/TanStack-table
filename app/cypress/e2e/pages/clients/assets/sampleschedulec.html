Each adventure consists of multiple adventurers. The adventurers will have a different permission based on each adventure

[EPIC] AdventureReq

Users can request to join an adventure based on its privacy settings and the owner or a moderator can either accept or reject the request.

A request has three states: 1.pending 2.accepted 3.rejected

Once accepted a new instance of an adventurer is created with the 'participant' role.

If rejected, the user can't join but the request won’t be deleted, meaning that either the owner or the moderators could just go back and accept the request and the user will be able to join.

Flow:

  • Users should be able to send a request to an adventure based on its privacy settings
  • The owner or the moderators could accept or reject a request

[EPIC] AdventureInv

Owners and moderators can invite a user and the invitee can either accept or reject the request.

An invite has three states: 1.pending 2.accepted 3.rejected

Once accepted a new instance of an adventurer is created with the 'participant' role.

If rejected, the user can't join but the request won’t be deleted, meaning that the invitee could just go back and accept the invite and join if he changes his mind.

Flow:

  • Users should be able to send a request to an adventure based on its privacy settings
  • The owner or the moderators could accept or reject a request
